# IDS Middleware

A FastAPI service that wraps the trained Random Forest model and exposes it over HTTP.
Built with **Clean Architecture** — domain logic is fully isolated from FastAPI, sklearn, and httpx.

---

## Quick Start

### 1. Export the model from the notebook

Run the last cell of `train.ipynb` — it saves four files into `model/`:

```plaintext
model/
├── rf_model.joblib           # trained Random Forest (300 trees)
├── feature_names.json        # ordered list of 41 features
├── dropped_correlated.json   # features removed during training
└── metadata.json             # performance metrics + config
```

### 2. Install dependencies

```bash
cd middleware
pip install -r requirements.txt
```

### 3. Start the service

```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

Interactive API docs: <http://localhost:8000/docs>

## Architecture

The codebase follows the **Dependency Rule**: inner layers never depend on outer layers.

```
┌──────────────────────────────────────────────────────────┐
│  api/                   (FastAPI — outermost layer)      │
│  ├── routers/           route handlers, thin controllers │
│  ├── schemas/           Pydantic request/response models │
│  └── dependencies.py   DI providers via FastAPI Depends  │
├──────────────────────────────────────────────────────────┤
│  infrastructure/        (concrete implementations)       │
│  ├── detection_service  ModelDetectionService            │
│  ├── alert_service      WebhookAlertService              │
│  ├── feature_pipeline   FeaturePipeline (all transforms) │
│  └── model_loader       loads .joblib + JSON artifacts   │
├──────────────────────────────────────────────────────────┤
│  domain/                (pure Python — no frameworks)    │
│  ├── entities.py        ThreatLevel, Prediction, etc.    │
│  └── interfaces.py      IDetectionService, IAlertService │
├──────────────────────────────────────────────────────────┤
│  core/                  (cross-cutting)                  │
│  └── config.py          Settings from env vars           │
└──────────────────────────────────────────────────────────┘
```

### Key design decisions

| Pattern | Where | Why |
| --- | --- | --- |
| Abstract ports (ABC) | `domain/interfaces.py` | Routers depend on interfaces, not concrete classes |
| Dependency injection | `api/dependencies.py` + FastAPI `Depends()` | Services are swappable without touching routes |
| App factory | `main.py` → `create_app()` | Isolates startup logic; testable without side effects |
| Domain entities | `domain/entities.py` | Business objects (Prediction, ThreatLevel) have no HTTP knowledge |
| Schema adapters | `responses.py` `from_entity()` | Converts domain objects → Pydantic responses at the boundary |

## Endpoints

| Method | Path | Tag | Description |
| --- | --- | --- | --- |
| `GET` | `/health` | System | Liveness check |
| `GET` | `/model/info` | System | Model metadata + performance metrics |
| `POST` | `/predict` | Detection | Classify one NetFlow record |
| `POST` | `/predict/batch` | Detection | Classify up to 1 000 records |
| `POST` | `/predict/explain` | Detection | Classify + top 5 features + summary |
| `GET` | `/alerts/config` | Alerts | Current thresholds + sliding-window stats |
| `POST` | `/alerts/test` | Alerts | Send a test payload to the webhook |

### Threat levels

| Level | P(Attack) |
| --- | --- |
| `LOW` | < 0.50 |
| `MEDIUM` | 0.50 – 0.69 |
| `HIGH` | 0.70 – 0.89 |
| `CRITICAL` | ≥ 0.90 |

## Alerting

The service fires webhook alerts on two conditions:

| Alert type | Trigger | Endpoint |
| --- | --- | --- |
| `HIGH_CONFIDENCE_ATTACK` | Single prediction with confidence ≥ threshold | `/predict`, `/predict/explain` |
| `HIGH_ATTACK_RATE` | Batch attack rate ≥ rate threshold | `/predict/batch` |

Configure via environment variables before starting the server:

```bash
export ALERT_WEBHOOK_URL=https://hooks.slack.com/services/...
export ALERT_CONFIDENCE_THRESHOLD=0.90   # default
export ALERT_RATE_THRESHOLD=0.50         # default (50 % of window)
export ALERT_WINDOW_SIZE=100             # default (sliding window length)
```

Other supported env vars:

```bash
export MODEL_DIR=/path/to/model          # default: ../model
export LOG_LEVEL=INFO                    # default
```

Test your webhook:

```bash
curl -X POST http://localhost:8000/alerts/test
```

## Integration Patterns

### Inline application middleware (Python)

```python
import httpx

async def ids_check(flow: dict) -> bool:
    async with httpx.AsyncClient() as client:
        r = await client.post("http://localhost:8000/predict", json=flow)
    return r.json()["prediction"] == 0  # True = safe to proceed
```

### Reverse proxy (Nginx mirror)

Use `mirror` directive to send a copy of each request's connection metadata to the IDS service without blocking the original request.

### NetFlow collector

Use **softflowd** or **nfcapd** to export NetFlow v9 records from a network interface, then forward to `/predict/batch`.

### Stream processing (Kafka)

Consume NetFlow records from a Kafka topic and call `/predict/batch` for low-latency real-time scoring at scale.
